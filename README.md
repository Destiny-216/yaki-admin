# 焼肉店向け管理システム（Yaki-Admin）

Vue 3 と Element Plus を使用して構築された、
飲食店の現場運営を想定し、**注文・予約・メニュー・売上状況を一元管理**できる
焼肉店向けの管理システム（管理画面）です。

店舗運営に必要な情報を可視化し、
日々のオペレーションやバックオフィス業務の効率化を目的として開発しました。

------

##  📖 プロジェクト概要

本プロジェクトは、飲食店の **業務フローを想定した管理システム** をテーマにした
**個人開発のポートフォリオ作品**です。

- フロントエンドとバックエンドを分離した構成
- 実際の店舗運営を意識したデータ構造・ステータス設計
- 管理画面としての使いやすさ・視認性を重視

をポイントとして設計・実装しています。

------

## ✨ 主な機能

### 📊 ダッシュボード（Dashboard）

- 本日の売上・注文数・予約数などのKPI表示
- 売上・注文数の推移をグラフで可視化

### 🧾 注文管理（Order Management）

- 注文一覧の表示
- 注文ステータス管理（`pending / paid / done / cancelled`）
- 注文詳細の確認・検索・フィルタリング

### 🍖 メニュー管理（Menu Management）

- メニューの追加・編集・削除（CRUD）
- カテゴリ管理、表示／非表示切り替え
- 販売ステータス管理（販売中／売切／非表示）
- おすすめ商品の設定

### 📅 予約管理（Reservation Management）

- 予約一覧の表示
- 予約ステータス更新
- 顧客名・電話番号による検索

### 🏪 店舗設定（Shop Settings）

- 店舗名・住所・電話番号・営業時間の管理
- 営業中／休業中の切り替え
- お知らせ文の更新

------

## 🛠 使用技術

### フロントエンド

- **Vue 3**（Composition API / `<script setup>`）
- **Vite**
- **Element Plus**
- **ECharts**（売上・注文推移の可視化）
- **Fetch API**

### バックエンド

- **PHP**（REST API）
- **MySQL**

### アーキテクチャ

- フロントエンド／バックエンド分離構成
- 管理画面（SPA）＋ API サーバー

🔗 **Demo**  
https://yaki-admin.vercel.app/dashboard

## 🚀 セットアップ方法（ローカル）

### 前提条件

- Node.js（v16以上推奨）
- PHP + MySQL 環境
- バックエンド API が `http://localhost:8000` で起動していること

### インストール

```bash
npm install
```

### 開発サーバー起動

```bash
npm run dev
```

起動後、以下のURLにアクセスしてください。

```
http://localhost:5173
```
